# UI 视觉规范与组件规范

## 0. 说明

- 本文颜色、字号、间距均为基于截图逆向得到的近似规范，用于高保真还原第一版。
- 由于当前没有设计源文件与移动端截图，移动端章节属于开发建议，不视为对标站事实。

## 3. 标题编排模式

### 模式 A：浅色英文描边字 + 中文主标题

- 用于首页“热销产品”这类中心对齐标题。
- 英文位于上方或后方，字号更大、颜色极浅、透明度低。
- 中文标题使用主色红。
- 适合品牌感更强的首页区块。

### 模式 B：英文小标题 + 中文大标题

- 用于产品系列介绍、公司简介、深色数据区。
- 英文小标题为灰色大写，字距略放大。
- 中文标题为黑色或白色粗体。
- 下方紧接一段较长说明文字。

### 模式 C：红色英文单词 + 中文标题

- 用于首页“News / 新闻中心”。
- 英文标题直接参与视觉主导，颜色为主色红。
- 中文标题为黑色粗体，形成杂志式标题层级。

## 4. 布局与间距

### 页面宽度

- 主内容容器建议 `1200px - 1280px`
- 页面两侧留白较大，整体偏传统企业官网的居中布局

### 区块上下间距

| 场景 | 建议值 |
| --- | --- |
| 首页大区块之间 | `96px - 120px` |
| 内页主内容上下边距 | `56px - 80px` |
| 卡片列表上下边距 | `32px - 48px` |
| 标题与内容间距 | `24px - 32px` |

### 卡片内间距

| 场景 | 建议值 |
| --- | --- |
| 产品卡 | `24px - 28px` |
| 新闻卡 | `24px - 32px` |
| 表单容器 | `24px - 32px` |
| Logo 宫格 | `20px - 24px` |

### 行距与段落

- 长正文建议行高 `1.85 - 1.95`
- 段落间距建议 `16px - 20px`
- 编号条款型内容可使用 `12px - 16px` 的段落间距

## 5. 按钮规范

### 主按钮（描边）

- 白底
- `1px` 主色边框
- 主色文字
- 高度建议 `40px - 46px`
- 内边距建议 `0 22px - 28px`
- 适用于“查看该系列产品”等轻跳转入口

### 主按钮（实心胶囊）

- 主色红底
- 白色文字
- 右侧附带白底圆形箭头
- 高度建议 `48px - 56px`
- 圆角建议 `999px`
- 适用于首页数据区和新闻区“探索详细”

### 文本链接按钮

- 用于新闻列表、详情关联入口
- 文字深色或主色
- 可配小箭头图标

### Hover 建议

- 描边按钮：边框和文字保持主色，背景可微弱染红 `2% - 4%`
- 实心按钮：整体上浮 `2px`，箭头圆点略向右移动
- 全站不宜使用夸张缩放与大阴影

## 6. 卡片与列表规范

### 产品卡

- 首页热销产品卡更像“白底信息板”，无明显阴影
- 图片为主体，文字位于上方或左上
- 卡片更依赖留白和分栏，而不是边框与装饰

### 产品系列图文块

- 采用 `50 / 50` 横向分栏
- 一侧是纯文本白底，一侧是大图
- 相邻区块左右镜像翻转，制造页面节奏

### 宫格卡片

- 荣誉、证书、设备、合作企业全部采用规整宫格
- 边框极浅，圆角很小或无圆角
- Logo 与证书类图片应在统一容器中居中

### 新闻卡

- 首页新闻模块卡片圆角明显，大于全站其他页面
- 列表页新闻条目采用左图右文的横向条目
- 详情页为内容主栏 + 右侧相关文章侧栏

### 分页

- 小型方形分页
- 当前页红底白字
- 非当前页白底浅灰边
- 页间距较紧凑

## 7. 图片比例建议

| 图片类型 | 建议比例 | 说明 |
| --- | --- | --- |
| 首页 Hero | `21:10` 左右 | 宽幅场景图，适合视频或轮播图首屏 |
| 内页 Banner | `1920x300` 左右 | 横向裁切稳定，标题覆盖左侧 |
| 产品主图 | `4:3` 或透明 PNG | 适合白底展示设备主体 |
| 产品系列配图 | `1:1` 到 `4:3` | 首页图文交错大图 |
| 新闻封面 | `4:3` 或 `16:10` | 列表页和详情页统一 |
| 交车现场图片 | `16:10` | 兼顾车体与横幅场景 |
| 荣誉/证书图 | `3:4` | 方便宫格统一 |
| 合作伙伴 Logo | `3:2` 容器自适应 | 白底居中展示 |
| 联系页地图图 | `16:9` | 横向信息展示更稳定 |

## 8. 留白节奏

- 首页采用“大留白 + 少装饰”的节奏，尤其产品系列区文字区域非常克制。
- 重要模块之间依靠大面积白底切换，不依赖复杂分割线。
- 深色数据区作为情绪转折，前后区块都需要较大缓冲间距。
- 内页整体比首页更传统，白底信息区居多，节奏稳定。

## 9. Hover / 动效 / 切换方式

### 已从截图确认的交互

- 顶部导航 `hover` 下拉展开
- 右侧浮动工具条 `hover` 左向展开提示信息
- 横向页签切换有明显当前态高亮

### 可安全还原的动效建议

- 导航下拉：`150ms - 220ms` 的淡入下滑
- 卡片入场：首屏以下区块可做轻微 `fade-up`
- 数字区：可做数字滚动，但截图无法证明其真实存在，建议作为可开关效果
- 图片 hover：仅做 `scale(1.02)` 或阴影轻微变化

## 10. 组件细则

### Header

- 白底固定高度导航
- Logo 左、菜单中右、语言切换最右
- 一级当前态使用整块红底

### 子导航 Tabs

- 位于 Banner 下、内容上
- 白底横向排列
- 当前项红底白字
- 未激活项为白底深灰字

### Sidebar

- 仅产品中心使用
- 标题块红底白字
- 列表项白底深灰字
- 当前项红底白字
- 下方叠加联系方式模块

### Footer

- 深灰底
- 左中右三段式：品牌联系 / 栏目导航 / 二维码
- 底部独立版权线

## 11. 响应式建议

> 以下内容不是截图事实，而是落地时的推荐规则。

### PC（`>=1280px`）

- 首页热销产品 `4` 列
- 产品系列图文 `2` 列对半
- 新闻首页 `3` 列混合布局

### 平板（`768px - 1279px`）

- 热销产品改为 `2` 列
- 产品系列图文改为上下堆叠
- 新闻卡保持 `2` 列
- 横向子导航允许横向滚动

### 手机（`<=767px`）

- 所有宫格改为 `1` 列或 `2` 列
- Banner 高度压缩，标题左下对齐
- 产品侧边栏收进抽屉或折叠筛选
- 浮动工具栏缩减为底部悬浮 CTA
- 联系表单与地图上下堆叠
