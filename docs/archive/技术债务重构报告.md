# 官网技术债务重构报告

> 分支：`chore/tech-debt-remediation`（35 commits，已推送 origin，未合并 main）
> 范围：suneng 官网 monorepo — frontend(Next.js) / backend(NestJS+Prisma) / admin(React+Vite)

## 一、背景与目标

起点状态：项目能跑，但带着一批已识别的安全/功能/结构债，且**零自动化测试、CI 只负责部署**。目标是：把债务清掉，并留下一个**干净、可继续开发**的底子。

方法论：先用多 agent 做**对抗式审计**（不采信原方案自述，逐条读码取证），把"清单"变成验证过的事实；再分批修，**每批做真实端到端验证**（curl/并发/爬取/哨兵注入，不是只过编译），独立提交，按依赖顺序串行。

## 五、结构 / 干净底子

- **测试基础设施**（最大缺口）：搭 jest（后端）+ vitest（前端），给安全关键单元（RBAC 默认拒绝、上传 magic-byte、sanitizer、登录锁定）写回归测试，CI 加 test step 门住合并。
- **CMS 解耦试点**：about 页接已有的 v1/about 接口 + 静态 fallback（products 因后端无接口排除）。决策为**维持静态站**——admin 只管少数动态内容，其余硬编码即设计。
- **删死副本树**：`coding/`(1.3G) + `project-handover/`(10M) 整套陈旧副本删除；先解除 `frontend/node_modules` 指向 coding/ 的软链陷阱并重装为真实目录。
- **admin 死代码下线**：knip 验证后删 44 个死文件（整套未接通的 CMS 脚手架 + 真编辑器死代码）+ 移除带 XSS CVE 的未用 quill/react-quill + 删假编辑器工具栏（源文件 77→35）。
- **CI 质量门**：新增 ci.yml，三包 lint+typecheck+test+build，PR/push 阻塞。
- **部署管道加固**：deploy.sh 改为部署前备份 → 迁移 → 启动 → `nginx -s reload`（修缓存旧 IP 的 502）→ 容器内真实 /api/health 检查（修原 `curl -f http://localhost` 对 301 假阳性）；deploy.yml 改走 deploy.sh 并自门禁（lint+tsc+test，门不过不部署）+ frozen-lockfile。
- **杂项**：优雅关闭（Nest 生命周期钩子，原为 no-op）· docker 日志上限（防撑满盘）· seed 生产破坏性守卫 · 修复 seed 跑不起来（缺 @types/bcryptjs）+ 补 editor 种子账号 · DEPLOY.md 回滚流程 + nginx 校验 · .env 文档 + 根聚合 lint/test 脚本 · uploads gitignore。
