## 1. 通用约束（每个 Prompt 默认适用，除非该批另有说明）

### 工作区与 git
- 开工前必须 `git status`。工作区不干净 → 只输出未提交文件清单并停止，等用户指令；**不得用 stash/reset/checkout/clean 去"弄干净"它**。
- 工作区干净后，**仅允许**：`git switch -c <本批分支>`（从 main 当前 HEAD 切）、在该分支上 `git add`/`commit` **本批自己产生的改动**。
- **禁止**：stash、reset(任何模式)、revert、rebase、merge、cherry-pick、切到/丢弃其他分支、pull/fetch、push/force-push、tag；任何部署(deploy.sh / 触发 CI)、重启服务、改服务器、改 CI secret。
- **数据库**：仅允许对本地 dev 库操作；`prisma migrate dev` 生成迁移文件可以，但 `migrate deploy` / `db push` 到任何非 dev 库一律禁止、需人工审批。
- 绝不覆盖、丢弃或提交不属于本批的改动。一批一分支一主题；提交按子项分多次，scope 不混。

### 只动正确的树
- 仓库存在重复副本：`coding/*` 和 `project-handover/*` 是旧副本（`frontend/node_modules` 还软链进 `coding/`）。所有改动**只许落在顶层 `backend/` `frontend/` `admin/`**，严禁编辑 `coding/*` 或 `project-handover/*`（改那里=静默无效）。

### 不改契约 / 不删功能 / 不改视觉
- 不改 API contract、不删功能、不改页面可见排版（除非该批明确要求）。

### 验收（0 测试，全手动）
- ⚠️ **"三包 lint+tsc+build" 今天到货即全绿**，只能抓编译/类型/lint 回归，**抓不到本轮几乎所有安全/行为类回归**（403、XSS 绕过、响应头、IP、幂等、ISR 模式、fetch-vs-fallback…）。每批除三包外，**必须执行该批指定的可执行验收脚本**（curl/crawl/并发），附 before/after。
- **三包命令**（顶层树；node 用 20 与 deploy 对齐；后端 tsc 前先 prisma generate；admin 用 `tsc -b` 不是 `--noEmit`）：
  ```bash
  pnpm --dir backend prisma:generate          # 必须最先
  pnpm --dir frontend lint && pnpm --dir frontend build
  pnpm --dir backend  lint && pnpm --dir backend exec tsc --noEmit -p tsconfig.json && pnpm --dir backend build
  pnpm --dir admin    lint && pnpm --dir admin exec tsc -b && pnpm --dir admin build
  ```
- 需起服务/种子的批次（②C/③/④A/④B/④D/⑤）：本地起 dev 栈（Postgres + `prisma migrate` + `seed`），**种子必须同时含一个 super_admin 和一个 editor**。
- 每批输出：修复清单、before/after 证据、验收脚本结果、**是否影响功能 YES/NO**。
- 明确标注 **Agent 验不了、需人/运维** 的部分（S1 线上、prod secret、prod 迁移、cookie 跨域、停机、副本数）。
